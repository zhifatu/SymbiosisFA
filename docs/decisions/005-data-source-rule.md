# 决策记录 005：数据来源规则

## 问题
五个场系统存在大量硬编码阈值和自创计算公式，
导致数学核心与场系统脱节，测试不可靠。

## 决策
1. 创建 `DataSource` 作为所有场的唯一数据入口
2. 所有数值阈值集中在 DataSource.config
3. 所有数学计算（矩阵、间接、张力）由 core/math/ 提供
4. 场系统只做三件事：接收输入、调用 DataSource、返回结果

## 收益
- 阈值统一管理，一处修改处处生效
- 数学核心可独立测试
- 场系统回归到“薄封装层”的原始定位
- 新场开发者：你不需要知道数学怎么算，只需要知道从哪里拿数据

## 被拒绝的方案
- 统一五个场的实现架构（抹杀本质差异）
- 保留各场自创公式（导致数学核心被架空）
